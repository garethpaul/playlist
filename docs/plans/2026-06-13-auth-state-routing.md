# Integration Auth State Routing

status: completed

## Context

The login and Beats views duplicate the decision that both Twitter and Beats
social-auth records are required before entering the playlist flow. The
roadmap calls for auth-state routing tests, but the current dependency-free
suite does not isolate this decision from Django or unavailable integrations.

## Priorities

1. Centralize the required-integration auth predicate without changing routes.
2. Cover complete, partial, empty, and malformed auth collections using the
   existing standard-library import stubs.
3. Preserve all external calls, POST protections, templates, and legacy
   compatibility boundaries.

## Requirements

- R1. Add a pure `has_required_auths` helper that accepts only dictionaries.
- R2. Return true only when both `twitter` and `beats` entries are truthy.
- R3. Use the helper in `login` to redirect complete auth state to `/beats`.
- R4. Use the same helper in `beats` to redirect incomplete auth state to
  `/login` before external clients are constructed.
- R5. Add dependency-free tests for complete, one-provider, empty, `None`, and
  non-dictionary values.
- R6. Enforce the implementation, tests, documentation, and completed evidence
  in the existing baseline checker.
